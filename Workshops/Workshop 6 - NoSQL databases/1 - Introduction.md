NoSQL databases is a collection of different types of databases as alternative for relational databases.

- Flexibel datamodel: No fixed schema like relational databases.
- Storage as JSON documents, key-value pairs, columns-orientated or as graph
- Horizontal schaalbaar (multiple servers (sharding / partitioning)

- High performance (fast read/write and low latency)
- Eventual consistency (CAP-theorema)
- Does not use SQL (or limited use of SQL)

- Suitable for specific use-cases
- Key-value: e.g. caching / session management (Redis)
- Document store: JSON storage (MongoDB, CouchDB)
- Column stores: big-data, log-data analytics (Cassandra, HBase)
- Graph databases: relaties en netwerken (social network / aanbevelingen) Neo4J
